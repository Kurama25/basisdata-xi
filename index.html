Mengenal SQL dan XAMPP

Apa itu SQL?

SQL (Structured Query Language) adalah bahasa pemrograman standar yang digunakan untuk mengelola dan memanipulasi database relasional.

SQL memungkinkan pengguna untuk membuat, membaca, memperbarui, dan menghapus data dalam database. SQL dikembangkan oleh IBM pada tahun 1970-an dan menjadi standar ANSI pada tahun 1986.

Apa itu XAMPP?

XAMPP adalah paket perangkat lunak web server gratis dan open source yang dikembangkan oleh Apache Friends. XAMPP menyediakan lingkungan pengembangan lokal yang mudah untuk membuat dan menguji aplikasi web sebelum dipublikasikan ke server produksi.

Kepanjangan XAMPP:

X - Cross Platform

XAMPP dapat berjalan di berbagai sistem operasi seperti Windows, Linux, dan macOS. Ini memberikan fleksibilitas bagi developer untuk bekerja di platform yang mereka sukai tanpa perlu mengubah konfigurasi aplikasi.

A - Apache

Apache HTTP Server adalah web server open source yang paling populer di dunia. Apache bertugas untuk menerima permintaan HTTP dari browser dan mengirimkan halaman web yang diminta. Apache mendukung berbagai bahasa pemrograman dan modul ekstensi.

M - MySQL / MariaDB

MySQL (atau MariaDB pada versi terbaru) adalah sistem manajemen database relasional yang digunakan untuk menyimpan dan mengelola data aplikasi web. MySQL menggunakan SQL (Structured Query Language) untuk operasi database seperti membuat, membaca, mengubah, dan menghapus data.

P - PHP

PHP (Hypertext Preprocessor) adalah bahasa pemrograman server-side yang digunakan untuk membuat halaman web dinamis. PHP dapat berinteraksi dengan database MySQL dan menghasilkan HTML yang dikirim ke browser pengguna. PHP sangat populer untuk pengembangan web karena mudah dipelajari dan memiliki komunitas yang besar.

P - Perl

Perl adalah bahasa pemrograman yang serbaguna dan kuat untuk pengembangan web. Meskipun tidak sepopuler PHP, Perl masih digunakan untuk scripting CGI dan pemrosesan teks. XAMPP menyertakan Perl untuk memberikan opsi tambahan bagi developer.

Keunggulan XAMPP:

  • ‚Ä¢ Instalasi mudah dan cepat (one-click installation)
  • ‚Ä¢ Gratis dan open source
  • ‚Ä¢ Terintegrasi dengan phpMyAdmin untuk manajemen database visual

Data Definition Language (DDL)

Pengertian DDL

Data Definition Language (DDL) adalah subset dari SQL yang digunakan untuk mendefinisikan dan mengelola struktur database. DDL memungkinkan kita untuk membuat, mengubah, dan menghapus objek database seperti database, tabel, indeks, dan view.

Perintah DDL bersifat auto-commit, artinya perubahan yang dilakukan akan langsung tersimpan secara permanen ke dalam database dan tidak dapat di-rollback (dibatalkan).

Pengelompokan SQL

DDL

Data Definition Language

CREATE, ALTER, DROP, RENAME

DML

Data Manipulation Language

INSERT, SELECT, UPDATE, DELETE

DCL

Data Control Language

GRANT, REVOKE

1. Membuat Database (CREATE DATABASE)

Penjelasan:

Perintah CREATE DATABASE digunakan untuk membuat database baru dalam sistem MySQL. Database adalah wadah yang menampung kumpulan tabel dan objek database lainnya. Sebelum membuat tabel, kita harus membuat database terlebih dahulu.

Contoh:

CREATE DATABASE db_perpustakaan;
Query OK, 1 row affected (0.02 sec)

Perintah SHOW DATABASES menampilkan daftar semua database yang ada dalam sistem MySQL.

SHOW DATABASES;

Menggunakan Database (USE):

Perintah USE digunakan untuk memilih database yang akan digunakan untuk operasi selanjutnya.

USE db_perpustakaan;

2. Membuat Tabel (CREATE TABLE)

Penjelasan:

Perintah CREATE TABLE digunakan untuk membuat tabel baru dalam database. Tabel terdiri dari kolom-kolom yang memiliki nama dan tipe data. Setiap kolom dapat memiliki constraint (aturan) seperti PRIMARY KEY, NOT NULL, DEFAULT, dan lainnya.

Komponen CREATE TABLE:

  • ‚Ä¢ PRIMARY KEY: Kolom yang menjadi identitas unik setiap baris
  • ‚Ä¢ AUTO_INCREMENT: Nilai otomatis bertambah untuk setiap record baru
  • ‚Ä¢ NOT NULL: Kolom harus diisi, tidak boleh kosong
  • ‚Ä¢ DEFAULT: Nilai default jika tidak diisi

Contoh:

CREATE TABLE buku ( id_buku INT PRIMARY KEY AUTO_INCREMENT, judul VARCHAR(100) NOT NULL, pengarang VARCHAR(50), penerbit VARCHAR(50), tahun_terbit INT, stok INT DEFAULT 0 );
Query OK, 0 rows affected (0.05 sec)

Melihat Struktur Tabel (DESCRIBE / DESC):

Perintah DESC atau DESCRIBE menampilkan struktur tabel beserta informasi setiap kolom.

3. Mengubah Struktur Tabel (ALTER TABLE)

Penjelasan:

Perintah ALTER TABLE digunakan untuk mengubah struktur tabel yang sudah ada. Dengan ALTER TABLE, kita dapat menambah kolom baru, mengubah tipe data kolom, mengubah nama kolom, atau menghapus kolom.

a. Menambahkan Kolom (ADD COLUMN):

Digunakan untuk menambahkan kolom baru ke dalam tabel yang sudah ada.

ALTER TABLE buku ADD COLUMN kategori VARCHAR(30);
Query OK, 0 rows affected (0.03 sec) Records: 0 Duplicates: 0 Warnings: 0

b. Mengubah Tipe Data Kolom (MODIFY COLUMN):

Digunakan untuk mengubah tipe data atau constraint dari kolom yang sudah ada.

ALTER TABLE buku MODIFY COLUMN stok SMALLINT;
Query OK, 0 rows affected (0.04 sec) Records: 0 Duplicates: 0 Warnings: 0

c. Menghapus Kolom (DROP COLUMN):

Digunakan untuk menghapus kolom dari tabel secara permanen beserta seluruh datanya.

ALTER TABLE buku DROP COLUMN kategori;
Query OK, 0 rows affected (0.03 sec) Records: 0 Duplicates: 0 Warnings: 0

4. Menghapus Tabel (DROP TABLE)

Penjelasan:

Perintah DROP TABLE digunakan untuk menghapus tabel beserta seluruh struktur dan datanya secara permanen dari database. Operasi ini tidak dapat di-undo (dibatalkan), jadi harus digunakan dengan sangat hati-hati.

‚ö†Ô∏è Peringatan: DROP TABLE akan menghapus tabel dan semua data secara permanen! Pastikan Anda sudah membackup data penting sebelum menjalankan perintah ini.

Contoh:

Query OK, 0 rows affected (0.02 sec)

Data Manipulation Language (DML)

Pengertian DML

Data Manipulation Language (DML) adalah subset dari SQL yang digunakan untuk memanipulasi data dalam tabel database. DML mencakup operasi untuk menambahkan (INSERT), mengambil (SELECT), mengubah (UPDATE), dan menghapus (DELETE) data.

Berbeda dengan DDL yang mengubah struktur database, DML bekerja pada level data di dalam tabel. Operasi DML dapat di-rollback (dibatalkan) jika menggunakan transaksi.

1. Perintah INSERT

Penjelasan:

Perintah INSERT INTO digunakan untuk menambahkan data baru (record/baris) ke dalam tabel. Ada dua cara penggunaan INSERT: dengan menyebutkan nama kolom atau tanpa menyebutkan nama kolom.

a. INSERT dengan Menyebutkan Kolom:

Metode ini lebih aman dan disarankan karena kita bisa memilih kolom mana saja yang akan diisi. Kolom yang tidak disebutkan akan diisi dengan nilai DEFAULT atau NULL.

INSERT INTO buku (judul, pengarang, penerbit, tahun_terbit, stok) VALUES ('Laskar Pelangi', 'Andrea Hirata', 'Bentang Pustaka', 2005, 15);
Query OK, 1 row affected (0.01 sec)

b. INSERT Multiple Rows (Banyak Data Sekaligus):

Untuk efisiensi, kita bisa menambahkan beberapa baris data sekaligus dalam satu perintah INSERT.

INSERT INTO buku (judul, pengarang, stok) VALUES ('Bumi Manusia', 'Pramoedya Ananta Toer', 10), ('Perahu Kertas', 'Dee Lestari', 12), ('5 cm', 'Donny Dhirgantoro', 20);
Query OK, 3 rows affected (0.02 sec) Records: 3 Duplicates: 0 Warnings: 0

2. Perintah SELECT

Penjelasan:

Perintah SELECT digunakan untuk mengambil dan menampilkan data dari tabel. SELECT adalah perintah yang paling sering digunakan dalam SQL. Kita bisa memfilter data dengan klausa WHERE, mengurutkan dengan ORDER BY, dan membatasi jumlah data dengan LIMIT.

a. Menampilkan Semua Data:

Menggunakan asterisk (*) untuk menampilkan semua kolom dari tabel.

SELECT * FROM buku;
id_buku judul pengarang penerbit tahun_terbit stok
1 Laskar Pelangi Andrea Hirata Bentang Pustaka 2005 15
2 Bumi Manusia Pramoedya A.T. Hasta Mitra 1980 10
3 Perahu Kertas Dee Lestari Bentang Pustaka 2009 12
3 rows in set (0.00 sec)

b. Menampilkan Kolom Tertentu:

Kita bisa memilih kolom-kolom spesifik yang ingin ditampilkan untuk efisiensi query.

SELECT judul, pengarang FROM buku;
judul pengarang
Laskar Pelangi Andrea Hirata
Bumi Manusia Pramoedya A.T.
2 rows in set (0.00 sec)

c. Menampilkan Data dengan Kondisi (WHERE):

Klausa WHERE digunakan untuk memfilter data berdasarkan kondisi tertentu. Hanya baris yang memenuhi kondisi yang akan ditampilkan.

SELECT * FROM buku WHERE pengarang = 'Andrea Hirata';
id_buku judul pengarang penerbit tahun_terbit stok
1 Laskar Pelangi Andrea Hirata Bentang Pustaka 2005 15
1 row in set (0.00 sec)

d. Pencarian dengan LIKE (Pattern Matching):

Operator LIKE digunakan untuk mencari data dengan pola tertentu. Simbol % mewakili nol atau lebih karakter, sedangkan _ mewakili satu karakter.

SELECT * FROM buku WHERE judul LIKE '%Pelangi%';
id_buku judul pengarang penerbit tahun_terbit stok
1 Laskar Pelangi Andrea Hirata Bentang Pustaka 2005 15
1 row in set (0.00 sec)

3. Perintah UPDATE

Penjelasan:

Perintah UPDATE digunakan untuk mengubah data yang sudah ada dalam tabel. Klausa WHERE sangat penting untuk menentukan baris mana yang akan diubah. Tanpa WHERE, semua baris dalam tabel akan diupdate!

a. Update Data Tertentu:

Mengubah data pada baris tertentu berdasarkan kondisi WHERE.

UPDATE buku SET stok = 25 WHERE judul = 'Laskar Pelangi';
Query OK, 1 row affected (0.01 sec) Rows matched: 1 Changed: 1 Warnings: 0

b. Update Multiple Kolom:

Kita bisa mengubah beberapa kolom sekaligus dalam satu perintah UPDATE.

UPDATE buku SET stok = 30, penerbit = 'Gramedia' WHERE id_buku = 3;
Query OK, 1 row affected (0.01 sec) Rows matched: 1 Changed: 1 Warnings: 0

‚ö†Ô∏è Peringatan: Selalu gunakan WHERE saat UPDATE. Tanpa WHERE, semua baris dalam tabel akan diubah!

4. Perintah DELETE

Penjelasan:

Perintah DELETE digunakan untuk menghapus data dari tabel. Seperti UPDATE, klausa WHERE sangat penting untuk menentukan baris mana yang akan dihapus. DELETE hanya menghapus data, bukan struktur tabel.

a. Delete Data Tertentu:

Menghapus baris data berdasarkan kondisi tertentu.

DELETE FROM buku WHERE id_buku = 5;
Query OK, 1 row affected (0.01 sec)

b. Delete dengan Kondisi Kompleks:

Kita bisa menggunakan berbagai operator perbandingan dalam kondisi WHERE.

DELETE FROM buku WHERE tahun_terbit < 2000;
Query OK, 1 row affected (0.01 sec)

‚ö†Ô∏è Peringatan: DELETE adalah operasi permanen dan tidak dapat di-undo! Selalu gunakan WHERE untuk menghindari penghapusan semua data. Backup data penting sebelum melakukan DELETE.

Data Control Language (DCL)

DCL digunakan untuk mengontrol hak akses pengguna terhadap database. Perintah utama: GRANT dan REVOKE.

Perintah GRANT

1. Membuat User Baru

CREATE USER 'pustakawan'@'localhost' IDENTIFIED BY 'password123';
Query OK, 0 rows affected (0.02 sec)

2. Memberikan Privilege SELECT

GRANT SELECT ON db_perpustakaan.buku TO 'pustakawan'@'localhost';
Query OK, 0 rows affected (0.01 sec)

3. Memberikan Multiple Privileges

GRANT SELECT, INSERT, UPDATE ON db_perpustakaan.buku TO 'pustakawan'@'localhost';
Query OK, 0 rows affected (0.01 sec)

4. Memberikan Semua Privilege

GRANT ALL PRIVILEGES ON db_perpustakaan.* TO 'admin_perpus'@'localhost';
Query OK, 0 rows affected (0.01 sec)

Jenis-jenis Privilege:

  • SELECT: Membaca data
  • INSERT: Menambah data
  • UPDATE: Mengubah data
  • DELETE: Menghapus data
  • ALL PRIVILEGES: Semua hak akses

Perintah REVOKE

1. Mencabut Privilege Tertentu

REVOKE INSERT ON db_perpustakaan.buku FROM 'pustakawan'@'localhost';
Query OK, 0 rows affected (0.01 sec)

2. Mencabut Semua Privilege

REVOKE ALL PRIVILEGES ON db_perpustakaan.* FROM 'pustakawan'@'localhost';
Query OK, 0 rows affected (0.01 sec)

3. Menghapus User

DROP USER 'pustakawan'@'localhost';
Query OK, 0 rows affected (0.01 sec)

Melihat Privilege

SHOW GRANTS FOR 'pustakawan'@'localhost';
Grants for pustakawan@localhost
GRANT USAGE ON *.* TO `pustakawan`@`localhost`
GRANT SELECT ON `db_perpustakaan`.`buku` TO `pustakawan`@`localhost`
2 rows in set (0.00 sec)

Menerapkan Perubahan:

FLUSH PRIVILEGES;
Query OK, 0 rows affected (0.01 sec)
